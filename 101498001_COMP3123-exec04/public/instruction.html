COMP3123 Exercise 4 - Express.js API

Available Endpoints

GET
/hello

Returns "Hello Express JS" as plain text.

Example: GET http://localhost:3000/hello
Response: Hello Express JS
GET
/user

Accepts query string parameters: ?firstname=<value>&lastname=<value>

If parameters are not provided, defaults to firstname = "Tigran" and lastname = "Khachaturyan"

Example: GET http://localhost:3000/user?firstname=John&lastname=Doe
Response: {"firstname": "John", "lastname": "Doe"}
POST
/user/:firstname/:lastname

Expects path parameters for firstname and lastname.

Example: POST http://localhost:3000/user/John/Doe
Response: {"firstname": "John", "lastname": "Doe"}
POST
/users

Expects a JSON array in the request body, each item with firstname and lastname.

Example Request Body:
[
  {"firstname": "Tigran", "lastname": "Khachaturyan"},
  {"firstname": "John", "lastname": "Doe"},
  {"firstname": "John", "lastname": "Rome"}
]

Response: Same array as input

Static File Serving

This file is served using Express.js static middleware from the /public folder.

You can access it at: http://localhost:3000/instruction.html

Testing the API

You can test these endpoints using:

  • Postman
  • curl commands
  • Browser (for GET requests)
  • Any HTTP client

Development

To run the development server with auto-restart:

npm run dev

To run the production server:

npm start
